📄 CSV Column Order (Row 1 = header, data from Row 2)
Col1: Question  |  Col2: A  |  Col3: B  |  Col4: C  |  Col5: D  |  Col6: CorrectAnswer  |  Col7: Explanation  |  Col8: Type  |  Col9: Tolerance  |  Col10: Unit  |  Col11: Accept  |  Col12: MRQ_AON  |  Col13: Pairs  |  Col14: Items
Columns 1–7 are the base format compatible with all older question banks. Columns 8–14 are optional — leave them blank for MCQ questions. Wrap all cells in double quotes if they contain commas.
Question Type Reference — Col8: Type
mcq Multiple Choice — one correct answer Default if Type column is empty
Col6 (CorrectAnswer): Write exactly A, B, C, or D — whichever option is correct.
Cols 2–5 (A B C D): All four option texts. Leave C and D blank only if your question genuinely has two options.
Cols 9–14: Leave all blank.
"What is the capital of Nigeria?","Lagos","Abuja","Kano","Ibadan","B","FCT moved to Abuja in 1991.","mcq",,,,
mrq Multiple Response — one or more correct answers (checkboxes)
Col6 (CorrectAnswer): List ALL correct option letters separated by commas — e.g. A,C or A,B,D. Order does not matter.
Col12 (MRQ_AON): Write true for all-or-nothing scoring (student must get every correct option to earn any mark). Write false or leave blank for partial credit (student earns a fraction for each correct option chosen).
Cols 9–11, 13–14: Leave blank.
"Which are products of photosynthesis?","Glucose","Water","Oxygen","Carbon dioxide","A,C","CO₂ is consumed, not produced.","mrq",,,,false,
tf True / False
Col2 (A): The label shown on the True button — usually just write True.
Col3 (B): The label shown on the False button — usually just write False.
Cols 4–5 (C, D): Leave blank — True/False only uses A and B.
Col6 (CorrectAnswer): Write A if the statement is True, or B if it is False.
Cols 9–14: Leave all blank.
"Lagos is the capital of Nigeria.","True","False","","","B","The capital is Abuja since 1991.","tf",,,,
short Short Answer / Fill-in-the-Gap — student types a word or phrase
Cols 2–5 (A B C D): Leave all blank — Short Answer has no options.
Col6 (CorrectAnswer): The primary correct answer. Grading is case-insensitive exact match.
Col11 (Accept): Optional. Additional accepted answers separated by the pipe symbol |. Use this for spelling variations, abbreviations, or alternate forms — e.g. H₂O|water|h2o.
Cols 9–10, 12–14: Leave blank.
"The chemical symbol for water is ___","","","","","H₂O","H₂O or water are both accepted.","short",,,,"H2O|water|h₂o",
numeric Numeric Input — student enters a number
Cols 2–5 (A B C D): Leave all blank.
Col6 (CorrectAnswer): The exact correct number — e.g. 9.81 or 42.
Col9 (Tolerance): Acceptable margin of error. Write 0 for exact match, or a number like 0.5 to accept answers within ±0.5 of the correct value. Leave blank for exact match.
Col10 (Unit): Optional display unit shown to the student — e.g. m/s², kg, °C.
Cols 11–14: Leave blank.
"Calculate the acceleration due to gravity (standard value)","","","","","9.81","g = 9.81 m/s²","numeric","0.05","m/s²",,
matching Matching — student pairs each left item to the correct right item
Col1 (Question): The question instruction — e.g. "Match each country to its capital city."
Cols 2–5 (A B C D): Leave all blank.
Col6 (CorrectAnswer): Leave blank — correctness is determined by the pairs in Col13.
Col13 (Pairs): The matching pairs written in JSON format. Each pair is {"left":"...","right":"..."} and all pairs are wrapped in a JSON array. Distractors (extra right-side options that do not match any left item) are appended as {"left":"DISTRACTOR","right":"Extra wrong option"} — the system recognises the keyword DISTRACTOR and uses only the right value as a decoy.
Cols 9–12, 14: Leave blank.
"Match each element to its symbol","","","","","","","matching","","","","","[{""left"":""Sodium"",""right"":""Na""},{""left"":""Potassium"",""right"":""K""},{""left"":""Iron"",""right"":""Fe""},{""left"":""DISTRACTOR"",""right"":""Au""}]",""
⚠️ In CSV, every double-quote inside a cell must be escaped as two double-quotes (""). The cell itself must be wrapped in double-quotes.
ordering Ordering / Sequencing — student drags items into the correct order
Col1 (Question): The instruction — e.g. "Arrange the following steps of mitosis in the correct order."
Cols 2–5 (A B C D): Leave all blank.
Col6 (CorrectAnswer): Leave blank — the correct order is defined by Col14.
Col14 (Items): All items written as a JSON array in their CORRECT order. The system automatically scrambles them for the student. Write each item as a string — e.g. ["Prophase","Metaphase","Anaphase","Telophase"]
Cols 9–13: Leave blank.
"Arrange the stages of mitosis in the correct order","","","","","","","ordering","","","","","","[""Prophase"",""Metaphase"",""Anaphase"",""Telophase""]"
⚠️ Same JSON escaping rule: double-quotes inside a CSV cell must be written as "".
Complete Question Type Reference — 17 Types, Free Rule-Based Scoring, No AI API
mcq — Multiple choice. Col2–5 = options, Col6 = A/B/C/D.
tf — True/False. Col2=True, Col3=False, Col6=A or B.
mrq — Multiple response. Col6 = sorted letters, e.g. A,C; Col12 controls partial/all-or-nothing.
short — Typed answer. Col6 = primary answer; Col11 = accepted alternatives with |.
numeric — Number input. Col6 = answer, Col9 = tolerance, Col10 = unit.
matching — Left/right pairs. Col13 = JSON pair array plus optional DISTRACTOR rows.
ordering — Sequencing. Col14 = JSON array in correct order.
cloze — Multi-blank. Question uses ___; Col14 = JSON answer list.
essay — Long response. Col11/Col14 = keywords/min words; teacher review recommended.
categorization — Assign item to category. Col14 = {"item":"...","category":"..."} array.
multi_numeric — Several numeric parts. Col14 = {"label":"x","answer":3,"tolerance":0} array.
assertion_reason — Assertion–Reason logic. Col6 = A/B/C/D/E; Col14 may contain assertion/reason JSON.
case_study — Passage/scenario MCQ. Col14 = {"passage":"..."}.
image_mcq — Diagram/image MCQ. Col11 or Col14 may contain image path/data URI.
matrix — Grid rows with shared choices. Col11 = choices like True|False; Col14 = row JSON.
hot_text — Select correct words/phrases. Col14 = text chunks with correct:true.
code — Code/SQL/algorithm response. Col10 = language; Col11/Col14 = keywords/tests; teacher review recommended.
JSON escaping rule: Whenever Col13 or Col14 contains JSON, wrap the whole CSV cell in double-quotes and write every inner double-quote as two double-quotes. Example: "[{""item"":""Sodium"",""category"":""Metal""}]".
No AI API: Essay and code answers use transparent keyword/minimum-word/test rules and should be reviewed by the teacher for high-stakes grading.
Prompt help: Use PROMPT_TEMPLATE.md to ask an AI assistant to generate a ready-to-upload CSV question bank manually.
✏️ Manual Entry — Type each question directly in the browser. Choose the question type first — the form fields change automatically to match the type. Required fields are marked. Explanation is always optional.
📊 Excel Column Order (Row 1 = header, data from Row 2)
A: Question  |  B: Option A  |  C: Option B  |  D: Option C  |  E: Option D  |  F: CorrectAnswer  |  G: Explanation  |  H: Type  |  I: Tolerance  |  J: Unit  |  K: Accept  |  L: MRQ_AON  |  M: Pairs  |  N: Items
Columns A–G are the base format. Columns H–N are optional — leave the cells empty for standard MCQ questions.
How to fill each column per question type
mcq (or blank) — Fill A–G normally. F = A/B/C/D. H may be blank or mcq.
mrq — F = comma list like A,C. H = mrq. L = true or false.
tf — B=True, C=False, D/E blank. F = A or B. H = tf.
short — F = primary answer. K = alternates like H₂O|water. H = short.
numeric — F = number, I = tolerance, J = unit. H = numeric.
matching — M = JSON pair array. Use {"left":"DISTRACTOR","right":"..."} for decoys. H = matching.
ordering — N = JSON array in correct order. H = ordering.
cloze — Question uses ___; N = JSON answer list. H = cloze.
essay — K = keywords; N = JSON rubric with min_words and keywords. H = essay.
categorization — N = JSON item/category rows. H = categorization.
multi_numeric — N = JSON numeric parts. H = multi_numeric.
assertion_reason — F = A/B/C/D/E. N may contain assertion/reason JSON. H = assertion_reason.
case_study — B–E = options, F = A/B/C/D, N = {"passage":"..."}. H = case_study.
image_mcq — B–E = options, F = answer, K or N = image path/data URI. H = image_mcq.
matrix — K = shared choices like True|False, N = row JSON. H = matrix.
hot_text — N = chunks JSON with correct:true. H = hot_text.
code — J = language, K/N = keywords/tests. H = code; teacher review recommended.
⚠️ Excel Tips: For Matching and Ordering, type the JSON directly into the cell — do NOT wrap it in quotes in Excel (Excel handles the cell boundary automatically). Use regular double-quotes inside the JSON. Save as .xlsx, not .xls where possible.
📄 PDF Question Format Guide
The PDF parser reads text extracted from your PDF and recognises the most common structured types by their tags. For all 17 enterprise types, CSV/XLSX/manual entry is recommended because JSON schemas are more reliable there. Use a text-based PDF (not a scanned image). If your PDF was exported from Word or Google Docs, it will work perfectly.
Format per question type
⊙ MCQ — Multiple Choice
1. What is 2 + 2?
A) 3
B) 4
C) 5
D) 6
Answer: B
Explanation: 2 + 2 = 4.
Rules: Number each question (1. or 1)). Options must start with A) B) C) D). Answer line must start with Answer: followed by A, B, C, or D. Explanation line is optional.
☑ MRQ — Multiple Response
2. Which are products of photosynthesis? [MRQ]
A) Glucose
B) Water
C) Oxygen
D) Carbon dioxide
Answer: A,C
Explanation: Glucose and Oxygen are produced.
Add [MRQ] anywhere in the question text. Answer line lists all correct letters separated by commas: Answer: A,C
⟺ True / False
3. Lagos is the capital of Nigeria. [TF]
A) True
B) False
Answer: B
Explanation: The capital is Abuja since 1991.
Add [TF] anywhere in the question text. Option A must be the True label, B the False label. Answer is A (True) or B (False).
✏ Short Answer / Fill-in-the-Gap
4. The chemical symbol for water is ___. [SHORT]
Answer: H₂O
Accept: H2O|water|h₂o
Explanation: Water is H₂O.
Add [SHORT] in the question text. No A/B/C/D options needed. Answer line gives the primary correct answer. Optional Accept: line lists alternate answers separated by |.
# Numeric Input
5. Calculate the standard acceleration due to gravity. [NUMERIC]
Answer: 9.81
Tolerance: 0.05
Unit: m/s²
Explanation: g = 9.81 m/s² at sea level.
Add [NUMERIC] in the question text. No A/B/C/D options. Answer = correct number. Optional Tolerance: line gives the ± margin. Optional Unit: line gives the display unit.
⇌ Matching
6. Match each element to its chemical symbol. [MATCHING]
Left: Sodium | Right: Na
Left: Potassium | Right: K
Left: Iron | Right: Fe
Distractor: Au
Explanation: Na=Sodium, K=Potassium, Fe=Iron.
Add [MATCHING] in the question text. Each pair on its own line: Left: [item] | Right: [match]. Optional distractor lines add extra right-side options that do not match any left item: Distractor: [wrong option]. No A/B/C/D or Answer line needed.
⇅ Ordering / Sequencing
7. Arrange the stages of mitosis in the correct order. [ORDERING]
Item: Prophase
Item: Metaphase
Item: Anaphase
Item: Telophase
Explanation: The correct sequence is P-M-A-T.
Add [ORDERING] in the question text. List each item on its own line starting with Item:. The items must be listed in the CORRECT order — the platform scrambles them for the student automatically. No A/B/C/D or Answer line needed.
⚠️ PDF Tips: Use a text-based PDF, not a scanned image (scanned PDFs cannot be parsed). Export from Word, Google Docs, or LibreOffice for best results. Do not mix question types on the same line. Each question must start on a new line with its number.

🔐 Step 1 — Enable Row Level Security (RLS) on All Tables

This is the master switch. Without this, table-level isolation is not enforced.

-- Enable RLS on every application table ALTER TABLE public.profiles ENABLE ROW LEVEL SECURITY; ALTER TABLE public.exams ENABLE ROW LEVEL SECURITY; ALTER TABLE public.results ENABLE ROW LEVEL SECURITY; ALTER TABLE public.students ENABLE ROW LEVEL SECURITY;
🔒 Step 2 — Exams: Teachers Only See & Edit Their Own

Each teacher is completely isolated. They can only read, create, update, and delete their own exams.

-- Helper required before any admin-aware policy below. CREATE OR REPLACE FUNCTION public.is_platform_admin() RETURNS BOOLEAN LANGUAGE SQL SECURITY DEFINER STABLE SET search_path = public, pg_temp AS $$ SELECT COALESCE(( SELECT (p.is_admin=true OR p.role='admin') AND p.status='active' FROM public.profiles p WHERE p.id=auth.uid() LIMIT 1 ), false) OR lower(COALESCE(auth.jwt()->>'email','')) = lower('buildingmyictcareer@gmail.com'); $$; -- Exams: teachers manage their own exams; platform admins can supervise all. -- Students load exams through get_public_exam_by_code(), not broad anonymous SELECT. DROP POLICY IF EXISTS "Students can read exams by code" ON public.exams; DROP POLICY IF EXISTS "Teachers select own exams" ON public.exams; DROP POLICY IF EXISTS "Teachers insert own exams" ON public.exams; DROP POLICY IF EXISTS "Teachers update own exams" ON public.exams; DROP POLICY IF EXISTS "Teachers delete own exams" ON public.exams; CREATE POLICY "Teachers select own exams" ON public.exams FOR SELECT TO authenticated USING (auth.uid() = teacher_id OR public.is_platform_admin()); CREATE POLICY "Teachers insert own exams" ON public.exams FOR INSERT TO authenticated WITH CHECK (auth.uid() = teacher_id OR public.is_platform_admin()); CREATE POLICY "Teachers update own exams" ON public.exams FOR UPDATE TO authenticated USING (auth.uid() = teacher_id OR public.is_platform_admin()) WITH CHECK (auth.uid() = teacher_id OR public.is_platform_admin()); CREATE POLICY "Teachers delete own exams" ON public.exams FOR DELETE TO authenticated USING (auth.uid() = teacher_id OR public.is_platform_admin());
🔒 Step 3 — Results: Full RLS Policies (FIXED — Run This Version)

⚠️ The standard subquery approach (SELECT teacher_id FROM exams WHERE id = exam_id) causes a recursive RLS deadlock because the exams table also has RLS enabled. This version uses a SECURITY DEFINER helper function that bypasses RLS safely for the lookup only.

-- Results: create helper functions BEFORE policies, then create safe policies. CREATE OR REPLACE FUNCTION public.get_exam_teacher_id(p_exam_id UUID) RETURNS UUID LANGUAGE SQL SECURITY DEFINER STABLE SET search_path = public, pg_temp AS $$ SELECT teacher_id FROM public.exams WHERE id = p_exam_id LIMIT 1; $$; CREATE OR REPLACE FUNCTION public.is_exam_open_for_submission(p_exam_id UUID) RETURNS BOOLEAN LANGUAGE SQL SECURITY DEFINER STABLE SET search_path = public, pg_temp AS $$ SELECT EXISTS ( SELECT 1 FROM public.exams e WHERE e.id = p_exam_id AND e.is_archived = false AND e.is_open = true AND (e.start_at IS NULL OR e.start_at <= NOW()) AND (e.close_at IS NULL OR e.close_at > NOW()) ); $$; DROP POLICY IF EXISTS "Teachers select own results" ON public.results; DROP POLICY IF EXISTS "Students can submit results" ON public.results; DROP POLICY IF EXISTS "Teachers insert own results" ON public.results; DROP POLICY IF EXISTS "Teachers update own results" ON public.results; DROP POLICY IF EXISTS "Teachers delete own results" ON public.results; CREATE POLICY "Teachers select own results" ON public.results FOR SELECT TO authenticated USING (auth.uid() = public.get_exam_teacher_id(exam_id) OR public.is_platform_admin()); CREATE POLICY "Students can submit results" ON public.results FOR INSERT TO anon WITH CHECK (public.is_exam_open_for_submission(exam_id)); CREATE POLICY "Teachers insert own results" ON public.results FOR INSERT TO authenticated WITH CHECK (auth.uid() = public.get_exam_teacher_id(exam_id) OR public.is_platform_admin()); CREATE POLICY "Teachers update own results" ON public.results FOR UPDATE TO authenticated USING (auth.uid() = public.get_exam_teacher_id(exam_id) OR public.is_platform_admin()) WITH CHECK (auth.uid() = public.get_exam_teacher_id(exam_id) OR public.is_platform_admin()); CREATE POLICY "Teachers delete own results" ON public.results FOR DELETE TO authenticated USING (auth.uid() = public.get_exam_teacher_id(exam_id) OR public.is_platform_admin());

👨‍🎓 Step 6 — Create Students Table

This table stores your class roster for registered-mode exams. Each teacher only sees their own students.

-- Registered-student roster table and secure teacher/admin policies. CREATE TABLE IF NOT EXISTS public.students ( id UUID PRIMARY KEY DEFAULT gen_random_uuid(), teacher_id UUID NOT NULL REFERENCES auth.users(id) ON DELETE CASCADE, full_name TEXT NOT NULL, student_id TEXT NOT NULL, class TEXT NOT NULL DEFAULT '', created_at TIMESTAMPTZ DEFAULT NOW(), UNIQUE(teacher_id, student_id) ); DROP POLICY IF EXISTS "Anyone can verify student ID" ON public.students; DROP POLICY IF EXISTS "Teachers select own students" ON public.students; DROP POLICY IF EXISTS "Teachers insert own students" ON public.students; DROP POLICY IF EXISTS "Teachers update own students" ON public.students; DROP POLICY IF EXISTS "Teachers delete own students" ON public.students; CREATE POLICY "Teachers select own students" ON public.students FOR SELECT TO authenticated USING (auth.uid() = teacher_id OR public.is_platform_admin()); CREATE POLICY "Teachers insert own students" ON public.students FOR INSERT TO authenticated WITH CHECK (auth.uid() = teacher_id OR public.is_platform_admin()); CREATE POLICY "Teachers update own students" ON public.students FOR UPDATE TO authenticated USING (auth.uid() = teacher_id OR public.is_platform_admin()) WITH CHECK (auth.uid() = teacher_id OR public.is_platform_admin()); CREATE POLICY "Teachers delete own students" ON public.students FOR DELETE TO authenticated USING (auth.uid() = teacher_id OR public.is_platform_admin());
⚠️ Step 7 — Fix Existing Teachers Already on the Platform

If teachers registered BEFORE the profiles table was created, they have no profile row and will be blocked on login. Run this SQL to create active profiles for all existing auth users. Change 'buildingmyictcareer@gmail.com' to your actual admin email.

-- Migrate existing Auth users into profiles. Change the admin email if needed. INSERT INTO public.profiles (id, email, full_name, role, is_admin, status) SELECT u.id, u.email, COALESCE(u.raw_user_meta_data->>'full_name', u.raw_user_meta_data->>'display_name', split_part(u.email, '@', 1)), CASE WHEN lower(u.email) = lower('buildingmyictcareer@gmail.com') THEN 'admin' ELSE 'teacher' END, CASE WHEN lower(u.email) = lower('buildingmyictcareer@gmail.com') THEN true ELSE false END, 'active' FROM auth.users u WHERE NOT EXISTS (SELECT 1 FROM public.profiles p WHERE p.id = u.id) ON CONFLICT (id) DO NOTHING;

✅ Step 8b — Add Scoring Count Columns (Fixes Student/Teacher Score Mismatch)

These three columns store the correct, wrong, and skipped counts computed by the student's browser at submission time. This is the ground truth — the teacher dashboard reads these directly instead of re-scoring from answers_data, which could produce different results if the question bank was edited after students submitted. Run this if your teacher dashboard shows different wrong counts than students see.

-- Accurate result columns for objective + partial-credit scoring. ALTER TABLE public.results ADD COLUMN IF NOT EXISTS correct_count INTEGER DEFAULT NULL; ALTER TABLE public.results ADD COLUMN IF NOT EXISTS wrong_count INTEGER DEFAULT NULL; ALTER TABLE public.results ADD COLUMN IF NOT EXISTS skipped_count INTEGER DEFAULT NULL; ALTER TABLE public.results ADD COLUMN IF NOT EXISTS attempt_number INTEGER DEFAULT 1; ALTER TABLE public.results ALTER COLUMN score TYPE NUMERIC(10,2) USING score::NUMERIC;
👑 Step 9 — Admin RPC Functions (Run in Supabase SQL Editor)

These functions let the admin panel see ALL teachers' data across RLS boundaries. The DROP … CASCADE lines remove any old version first — this fixes the "cannot change return type" error.

-- STEP 9: PUBLIC STUDENT RPC FUNCTIONS -- ═══════════════════════════════════════════════════════════════════ DROP FUNCTION IF EXISTS public.get_public_exam_by_code(TEXT) CASCADE; DROP FUNCTION IF EXISTS public.verify_student_for_exam(UUID, TEXT) CASCADE; DROP FUNCTION IF EXISTS public.get_exam_attempt_count(UUID, TEXT, TEXT, TEXT) CASCADE; CREATE OR REPLACE FUNCTION public.get_public_exam_by_code(p_code TEXT) RETURNS TABLE ( id UUID, code TEXT, subject TEXT, duration INTEGER, attempt_limit INTEGER, select_count INTEGER, is_open BOOLEAN, exam_mode TEXT, negative_mark NUMERIC, release_results BOOLEAN, instructions TEXT, start_at TIMESTAMPTZ, close_at TIMESTAMPTZ, csv_data JSONB, created_at TIMESTAMPTZ, updated_at TIMESTAMPTZ ) LANGUAGE SQL SECURITY DEFINER STABLE SET search_path = public, pg_temp AS $$ SELECT e.id, e.code, e.subject, e.duration, e.attempt_limit, e.select_count, e.is_open, e.exam_mode, e.negative_mark, e.release_results, e.instructions, e.start_at, e.close_at, CASE WHEN e.start_at IS NOT NULL AND e.start_at > NOW() THEN '[]'::jsonb ELSE e.csv_data END AS csv_data, e.created_at, e.updated_at FROM public.exams e WHERE upper(e.code) = upper(trim(p_code)) AND e.is_archived = false AND e.is_open = true AND (e.close_at IS NULL OR e.close_at > NOW()) LIMIT 1; $$; CREATE OR REPLACE FUNCTION public.verify_student_for_exam(p_exam_id UUID, p_student_id TEXT) RETURNS TABLE ( full_name TEXT, student_id TEXT, class TEXT ) LANGUAGE SQL SECURITY DEFINER STABLE SET search_path = public, pg_temp AS $$ SELECT s.full_name, s.student_id, s.class FROM public.students s JOIN public.exams e ON e.teacher_id = s.teacher_id WHERE e.id = p_exam_id AND e.is_archived = false AND upper(s.student_id) = upper(trim(p_student_id)) LIMIT 1; $$; CREATE OR REPLACE FUNCTION public.get_exam_attempt_count( p_exam_id UUID, p_student_name TEXT, p_student_class TEXT, p_student_id_ref TEXT DEFAULT '' ) RETURNS INTEGER LANGUAGE SQL SECURITY DEFINER STABLE SET search_path = public, pg_temp AS $$ SELECT COUNT(*)::INTEGER FROM public.results r WHERE r.exam_id = p_exam_id AND ( (COALESCE(trim(p_student_id_ref), '') <> '' AND upper(COALESCE(r.student_id_ref, '')) = upper(trim(p_student_id_ref))) OR (lower(trim(r.student_name)) = lower(trim(p_student_name)) AND lower(trim(r.student_class)) = lower(trim(p_student_class))) ); $$;
